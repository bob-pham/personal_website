Cornucopia

Project Overview

Cornucopia is intended to be an app which scans user grocery reciepts to help manager kitchen pantries and minimize food waste

Development/Implementation

As mentioned above, Cornucopia scans user grocery receipts and indexes products. This is accomplished using the OpenCV library, with text recognition being done using PyTesseract. The OpenCV library is primarily used to hold the image and trim it into the minimum size and resolution that PyTesseract can recognize. The text that PyTessesract outputs are then filtered using a combination of Python functions mutating strings and Regex. Data is parsed into relevant objects once all irrelevant information is removed, and database storage is currently being developed.

Cornucopia is being developed using test-driven development. To test the accuracy of reading and filtering text, I created a custom unit testing framework on top of Python's unittest. The framework tests the text by comparing what is read and filtered with what the tester provides in a .txt document. The tester provides an accuracy score, and if the score calculated by the framework is lower than the provided score, the test fails. Accuracy is determined by looking at the differences in frequency of the characters that appear in the document and sentence structure.

A web scraper is currently being developed to find product recalls. These would then be put into a database, which can be searched through to determine if any user products have been recalled.

The front is currently being developed; however, the backend is still the priority. Django is being used to connect our backend to its eventual frontend. The frontend is intended to use React Native, just for practice, given its incredible popularity. Both of these technologies are currently being researched.
